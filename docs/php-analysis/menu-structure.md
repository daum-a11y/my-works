# 메뉴/화면 구조

## 1. 전체 IA

```text
로그인
└─ 로그인 성공
   ├─ Dashboard
   ├─ 업무보고
   ├─ 프로젝트 관리
   │  ├─ 모니터링 트래킹
   │  └─ 프로젝트 등록 & 리스트
   ├─ 기타
   │  ├─ 앱 운영정보
   │  └─ 업무보고 검색
   └─ 통계
      ├─ QA
      └─ 모니터링
```

## 2. 상단 메뉴

| 메뉴 | 동작 | 파일 |
| --- | --- | --- |
| 개인정보 수정 | 내 비밀번호 변경 화면 로드 | `pages/userset.php` |
| 로그아웃 | 세션 종료 후 재진입 | `login/clear.php` |

## 3. 좌측 사이드바 구조

| 1Depth | 2Depth | 화면 파일 | 비고 |
| --- | --- | --- | --- |
| Dashboard | - | `pages/dashboard.php` | 기본 랜딩 |
| 업무보고 | - | `pages/report.php` | 입력 + 내역 조회 |
| 프로젝트 관리 | 모니터링 트래킹 | `pages/track.php` | 트래킹 수정 |
| 프로젝트 관리 | 프로젝트 등록 & 리스트 | `pages/project.php` | 프로젝트/페이지 CRUD |
| 기타 | 앱 운영정보 | `pages/appinfo.php` | 앱 운영 상태 관리 |
| 기타 | 업무보고 검색 | `pages/report_personal.php` | 개인 업무 조회/다운로드 |
| 통계 | QA | `pages/stati_qa.php` | QA 통계 |
| 통계 | 모니터링 | `pages/stati_mo.php` | 모니터링 통계 |

## 4. 숨김/메뉴 미노출 화면

다음 화면 파일은 저장소에 존재하지만 현재 `nav.php` 메뉴에서는 직접 노출되지 않습니다.

| 화면 파일 | 목적 | 현재 상태 해석 |
| --- | --- | --- |
| `pages/allreport.php` | 전체 리소스 검색/다운로드 | 관리용 또는 숨김 기능 |
| `pages/type.php` | 업무 Type 관리 | 기준정보 관리 화면 |
| `pages/service_group.php` | 서비스 그룹 관리 | 기준정보 관리 화면 |
| `pages/members.php` | 계정관리 | 관리자 성격 화면 |
| `pages/new_member.php` | 신규 계정 생성 | 구형 계정 생성 화면 |
| `pages/test.php` | 테스트 코드 | 운영 대상 아님 |

## 5. 메뉴 전환 방식

- 브라우저 URL 라우팅을 쓰지 않습니다.
- [`webapp/js/link.js`](../../webapp/js/link.js)가 `#page-wrapper`에 `pages/*.php`를 `load()`합니다.
- 따라서 메뉴 클릭은 "페이지 이동"보다는 "중앙 작업영역 교체"에 가깝습니다.

## 6. 사용자 관점 탐색 흐름

### 6.1 일반적인 일일 운영 흐름

1. 로그인
2. Dashboard에서 진행 현황 확인
3. 업무보고 화면에서 업무 입력
4. 필요 시 프로젝트 관리 또는 트래킹 화면 이동
5. 업무보고 검색에서 기간별 이력 조회

### 6.2 관리자성 운영 흐름

1. 프로젝트 등록/수정
2. 프로젝트 페이지 추가/수정
3. 트래킹 상태 갱신
4. 앱 운영정보/Type/서비스 기준정보 관리
5. 전체 리소스 검색 다운로드

## 7. 메뉴 구조 해석 메모

- 공개 운영 메뉴는 `Dashboard / 업무보고 / 프로젝트 / 기타 / 통계` 5개 축으로 충분히 설명됩니다.
- 숨김 화면들은 유지보수 시 중요한 힌트이므로 문서에서 제외하면 안 됩니다.
- 이 시스템의 실질적 정보 구조는 메뉴보다 데이터 흐름 기준으로 `업무보고`, `프로젝트`, `트래킹`, `통계`, `기준정보` 5개 도메인으로 정리하는 편이 이해하기 쉽습니다.
