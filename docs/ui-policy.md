# UI 정책

## 화면 대응 범위

- 현재 서비스는 **데스크톱 전용(Desktop Only)**으로 운영됩니다.
- 모바일/태블릿 반응형 별도 레이아웃은 현재 지원하지 않습니다.
- 반응형 대응이 필요할 경우 `레이아웃 정책`과 `전역/공통 스타일`을 함께 정비하는 별도 작업이 필요합니다.

## 프로젝트 관리 화면 규칙

- `프로젝트 관리` 목록 화면은 `통계` 화면의 필터 문법을 기준으로 맞춥니다.
- 필터는 `라벨 + 입력`, 우측 끝 `검색/초기화` 액션 조합을 기본으로 하며, 버튼 높이와 정렬은 `src/features/stats/shared.module.css` 기준을 따릅니다.
- 목록 표는 `업무보고 검색`, `통계` 표와 같은 밀도와 헤더 문법을 사용합니다.
- 표 셀에서 의미 없는 강조 표현을 금지합니다. 플랫폼 배지, 프로젝트명 볼드, 전체 URL 노출 같은 일회성 표현을 넣지 않습니다.
- 링크가 필요한 셀은 전체 URL 문자열 대신 `링크` 같은 짧은 액션 라벨로 노출합니다.
- 목록 화면은 `조회` 역할만 가집니다. `추가`, `수정`은 별도 페이지에서 처리합니다.
- 프로젝트 화면 전용 버튼 색을 만들지 않습니다. 주 버튼, 보조 버튼, 위험 버튼은 기존 공용 역할색을 그대로 사용합니다.
