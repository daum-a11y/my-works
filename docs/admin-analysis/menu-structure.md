# 메뉴/화면 구조

## 1. 전체 IA

```text
로그인
└─ 로그인 성공
   ├─ Dashboard
   ├─ 보고
   │  ├─ 업무보고
   │  └─ 기간검색
   ├─ 리소스 (관리자 노출)
   │  ├─ 요약
   │  ├─ 타입별 요약
   │  ├─ 그룹별 요약
   │  └─ 월간 리소스
   ├─ 검색 (관리자 노출)
   ├─ 관리 (관리자 노출)
   │  ├─ 요약
   │  ├─ 업무 타입
   │  │  ├─ 관리모드 (현재 스펙아웃)
   │  │  └─ 유효성검증모드
   │  ├─ 서비스 그룹
   │  │  ├─ 관리모드 (현재 스펙아웃)
   │  │  └─ 유효성검증모드
   │  ├─ 사용자
   │  └─ 아지트QA알리미 (현재 스펙아웃)
   └─ 프로필
```

## 2. 상단 메뉴

| 메뉴 | 라우트 | 노출 조건 | 비고 |
| --- | --- | --- | --- |
| 홈 | `/` | 로그인 사용자 | Dashboard |
| 보고 | `/report` | 로그인 사용자 | 개인 업무/개인 기간검색 |
| 리소스 | `/resource` | `memberLv > 0` | 관리자성 집계 화면 |
| 검색 | `/search` | `memberLv > 0` | 전체 업무 검색/수정 |
| 관리 | `/admin` | `memberLv > 0` | 기준정보/멤버 관리 |
| 프로필 | `/profile` | 로그인 사용자 | 비밀번호 변경 포함 |
| 로그아웃 | 버튼 | 로그인 사용자 | 토큰 제거 후 로그인 화면 |

## 3. 보고 영역 구조

| 1Depth | 2Depth | 라우트 | 목적 |
| --- | --- | --- | --- |
| 보고 | 업무보고 | `/report/personal/:type?` | 본인 일일 업무 입력/수정/삭제 |
| 보고 | 기간검색 | `/report/search` | 본인 기간 검색/엑셀 다운로드 |

구조 메모:

- `업무보고` 화면에서 일자 이동, 직접 입력, 오버헤드 자동 보충이 가능합니다.
- `기간검색`은 본인 데이터만 검색합니다.
- 현재 `Report.vue`는 `memberLv > 0`일 때만 실제 화면을 렌더링하므로, 상단 메뉴 노출 규칙과 불일치가 있습니다.

## 4. 리소스 영역 구조

| 1Depth | 2Depth | 라우트 | 목적 |
| --- | --- | --- | --- |
| 리소스 | 요약 | `/resource/summary` | 일간 작성현황 + 월간 개인 캘린더 |
| 리소스 | 타입별 요약 | `/resource/type` | 기간 누적 type 기준 MM 요약 |
| 리소스 | 그룹별 요약 | `/resource/svc` | 기간 누적 서비스 기준 MM 요약 |
| 리소스 | 월간 리소스 | `/resource/month/:type?` | 특정 월 상세 MM 보고서 |

## 5. 관리 영역 구조

| 1Depth | 2Depth | 3Depth | 라우트 | 목적 |
| --- | --- | --- | --- | --- |
| 관리 | 요약 | - | `/admin/summary` | 현재는 placeholder |
| 관리 | 업무 타입 | 관리모드 | `/admin/type` | `TYPE_TBL` CRUD |
| 관리 | 업무 타입 | 유효성검증모드 | `/admin/type` 내부 토글 | `TASK_TBL` orphan type bulk 보정 |
| 관리 | 서비스 그룹 | 관리모드 | `/admin/group` | `SVC_GROUP_TBL` CRUD |
| 관리 | 서비스 그룹 | 유효성검증모드 | `/admin/group` 내부 토글 | `TASK_TBL` orphan svc bulk 보정 |
| 관리 | 사용자 | - | `/admin/member` | `USER_TBL` CRUD, PW 초기화 |
| 관리 | 아지트QA알리미 | - | `/admin/agitnoti` | 현재 스펙아웃 |

## 6. 검색 영역 구조

| 화면 | 라우트 | 목적 |
| --- | --- | --- |
| 전체 검색 | `/search` | 전체 업무 검색, 필터, 인라인 수정/삭제, 신규 행 추가, 엑셀 다운로드 |

## 7. 숨김/비활성 화면

다음 화면은 저장소에 존재하지만 현재 활성 사용자 흐름에서는 직접 노출되지 않거나 비활성 상태입니다.

| 화면/기능 | 파일 | 현재 상태 해석 |
| --- | --- | --- |
| 프로젝트 뷰 | `front/src/views/Project.vue` | 라우터에서 주석 처리되어 비활성 |
| 프로젝트 목록들 | `front/src/components/project/*` | `PJ_TBL` 조회/삭제 UI 흔적 |
| 프로젝트 생성 모달 | `front/src/components/project/ProjectModalCreate.vue` | 라우트 비활성 상태에서만 존재 |
| 아지트 QA알리미 | `front/src/views/admin/AdminAgitNoti.vue` | 라우트는 살아 있으나 현재 스펙아웃 |
| `v-suggest` 플러그인 | `front/src/assets/v-suggest/*` | 공용 플러그인 등록은 돼 있으나 주요 화면 사용 흔적 미약 |

## 8. 사용자 관점 탐색 흐름

### 8.1 일반 업무 흐름

1. 로그인
2. Dashboard에서 본인 월간 작성현황 확인
3. 보고 > 업무보고에서 일일 업무 입력
4. 필요 시 보고 > 기간검색에서 본인 이력 조회/엑셀 다운로드
5. 프로필에서 비밀번호 변경

### 8.2 관리자 운영 흐름

1. 검색에서 전체 업무 데이터 조회
2. 관리 > 업무 타입 / 서비스 그룹에서 마스터 관리
3. 유효성검증모드에서 잘못 누적된 `TASK_TBL` 값을 일괄 보정
4. 관리 > 사용자에서 계정 생성/수정/비밀번호 초기화
5. 리소스 화면에서 일/월/기간 누적 집계 확인

## 9. 메뉴 구조 해석 메모

- 이 시스템의 실질적 IA는 `입력`, `검색`, `집계`, `마스터 관리`, `정합성 보정` 5축으로 보는 편이 정확합니다.
- 다만 현재 범위 기준으로 `마스터 관리 CRUD`는 스펙아웃이고, `정합성 보정(valid)`만 실질 대상입니다.
- 프로젝트는 1차 구조에는 있었지만 현재 활성 기능으로 보지 않는 것이 맞습니다.
- 아지트 QA알리미는 문서상 존재를 남기되, 2차 개발 범위에서는 제외 항목으로 명시해야 합니다.
