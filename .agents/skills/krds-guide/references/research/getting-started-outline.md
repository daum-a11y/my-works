## 1. 디자이너 시작하기 핵심 구조

페이지는 다음 흐름으로 구성된다.

1. 튜토리얼
2. 라이브러리
3. 라이브러리 시작하기(Figma)
4. 토큰 사용하기

### 확인한 핵심 내용

- KRDS는 디자이너가 일관되고 재사용 가능한 UX를 구축하도록 리소스와 튜토리얼을 제공한다.
- 피그마 라이브러리는 스타일 가이드, 컴포넌트, 기본 패턴, 서비스 패턴으로 나뉘며 디자인 토큰으로 연결된다.
- Sketch/XD 리소스는 색상과 타이포그래피 등 스타일 요소 중심의 보조 리소스다.
- 전체 라이브러리 사용에는 Figma 사용이 권장된다.

### Figma 시작 절차 메모

- `Pretendard GOV` 서체를 먼저 다운로드한다.
- KRDS 커뮤니티의 Figma 파일을 연다.
- 좌측 사이드바에서 스타일 가이드, 컴포넌트, 패턴, 아이콘 구조를 확인한다.
- 우측 사이드바에서 로컬 베리어블과 로컬 스타일을 확인한다.
- Assets 패널에는 대표 컴포넌트만 보일 수 있으며, 전체 변형은 관련 페이지에서 확인한다.
- 구성 요소를 다른 파일에서 재사용하려면 라이브러리 게시 후 개별 파일에 추가해야 한다.

### 디자이너 토큰 메모

- 베리어블 컬렉션은 `primitive`, `semantic`, `mode`, `responsive`로 소개된다.
- `primitive`는 원시값, 나머지는 실제 UI에 적용되는 의미 기반 토큰 계층으로 설명된다.
- 선명한 화면 모드와 반응형은 베리어블을 통한 테마 전환 흐름으로 제시된다.
- 색상 스타일은 단색값 중심의 로컬 베리어블과 그라데이션 포함 로컬 스타일을 함께 본다.

## 2. 개발자 시작하기 핵심 구조

페이지는 다음 흐름으로 구성된다.

1. 튜토리얼
2. 폰트 사용
3. HTML Component Kit 설치
4. React / Vue 설치
5. 디자인 토큰 추출
6. 스타일 사용자화
7. JavaScript 사용하기
8. 디자인 토큰 수정

### 확인한 핵심 내용

- 개발자 온보딩의 중심은 `KRDS HTML Component Kit` 설치와 적용이다.
- 표준형 스타일은 `Pretendard GOV` 웹폰트 CDN 사용을 전제로 설명된다.
- HTML Component Kit은 `NPM`, `GitHub 다운로드`, `CDN` 경로를 제공한다.
- React/Vue는 별도 패키지 설치와 Storybook 문서 참조 흐름이다.

### 개발자 설치 메모

- NPM 설치 시 SCSS 참조 경로 지원이 충분하지 않다는 유의 사항이 있다.
- SCSS 사용 시에는 Kit 내부 `/resources` 전체를 프로젝트로 복제하고 참조 경로를 수동 재설정하는 방식을 권장한다.
- 별도 설치가 어려운 경우 CSS/JS 파일 직접 로드 방식도 제공된다.

### 개발자 토큰 메모

- 디자인 툴에서 추출한 JSON 토큰을 CSS variable 형식으로 변환해 컴포넌트 스타일링에 사용한다.
- KRDS는 SCSS를 기본 작업 흐름으로 설명하지만, 컴파일된 CSS 파일도 함께 제공한다.
- 토큰 수정은 원본 코드 직접 수정이 아니라 프로젝트 CSS/SCSS에서 CSS variable을 재선언하는 방식이 원칙이다.
- 토큰 수정 전에는 디자인 토큰 목록과 연관 영향 범위를 먼저 확인하도록 안내한다.

## 3. 스킬 반영 포인트

- 이 두 문서는 `스타일`, `컴포넌트`, `패턴`보다 앞단의 `도입/설치/자산 준비` 문맥을 다룬다.
- 따라서 `krds-guide`에는 별도 `getting-started.md` 라우팅 축을 두는 편이 맞다.
- 디자이너 질문은 Figma 라이브러리와 디자인 토큰 사용 흐름을 먼저 설명하고, 구체 구현 단계에서만 `style.md`나 `components.md`로 넘긴다.
- 개발자 질문은 Kit 설치 경로, 토큰 연계 방식, 원본 수정 금지 원칙을 먼저 설명하고, 세부 UI 구현 단계에서 도메인 참조를 추가로 읽는다.
