출처:

- 기본 패턴 요약: <https://www.krds.go.kr/html/site/global/global_summary.html>

  - 개인 식별 정보 입력: <https://www.krds.go.kr/html/site/global/global_01.html>
  - 서비스 패턴 방문: <https://www.krds.go.kr/html/site/service/service_01_01.html>

## 1. 기본 패턴 요약

요약 페이지가 `총 11건`으로 표기되지만 실제 노출 목록은 12개입니다.  
새로 추가된 모바일 항목이 카운터에 반영되지 않았을 가능성을 염두에 두는 편이 좋습니다.

### 1.1 세부 메뉴

- 개인 식별 정보 입력: <https://www.krds.go.kr/html/site/global/global_01.html>
- 도움: <https://www.krds.go.kr/html/site/global/global_02.html>
- 동의: <https://www.krds.go.kr/html/site/global/global_03.html>
- 목록 탐색: <https://www.krds.go.kr/html/site/global/global_04.html>
- 사용자 피드백: <https://www.krds.go.kr/html/site/global/global_05.html>
- 상세 정보 확인: <https://www.krds.go.kr/html/site/global/global_06.html>
- 오류: <https://www.krds.go.kr/html/site/global/global_07.html>
- 입력폼: <https://www.krds.go.kr/html/site/global/global_08.html>
- 첨부 파일: <https://www.krds.go.kr/html/site/global/global_09.html>
- 필터링·정렬: <https://www.krds.go.kr/html/site/global/global_10.html>
- 확인: <https://www.krds.go.kr/html/site/global/global_11.html>
- 모바일: <https://www.krds.go.kr/html/site/global/global_12.html>

### 1.2 각 패턴의 요약 의미

- 개인 식별 정보 입력:
  개인/단체 신원 및 기본 정보 입력
- 도움:
  서비스 사용법, UI 작동 방식, 플로 지원
- 동의:
  약관/절차/안내 확인 및 동의
- 목록 탐색:
  다수 항목 탐색과 항목 기반 작업
- 사용자 피드백:
  경험 평가, 제안, 불편 수집
- 상세 정보 확인:
  특정 주제의 상세 콘텐츠 전달
- 오류:
  실패 상황 전달과 복구 유도
- 입력폼:
  서버 전송용 입력 섹션
- 첨부 파일:
  다운로드 가능한 첨부 콘텐츠
- 필터링·정렬:
  데이터 집합 축소/재조직
- 확인:
  취소하기 어려운 행동 전 재검토
- 모바일:
  모바일 알림 중심 보조 패턴

## 2. 서비스 패턴 요약

서비스 패턴은 사용자 여정 기반 UX 가이드로 설명됩니다.

세부 메뉴:

- 방문: <https://www.krds.go.kr/html/site/service/service_01_01.html>
- 검색: <https://www.krds.go.kr/html/site/service/service_02_01.html>
- 로그인: <https://www.krds.go.kr/html/site/service/service_03_01.html>
- 신청: <https://www.krds.go.kr/html/site/service/service_04_01.html>
- 정책 정보 확인: <https://www.krds.go.kr/html/site/service/service_05_01.html>

서비스 패턴 설명의 핵심:

- 핵심 과업을 사용자 여정으로 설명
- 모범적 표준 프로토타입 제시
- 사용성, 접근성, 상호작용, 플랫폼 고려사항 기술
- 적용 수준을 `필수(Do) - 권장(Better) - 우수(Best)`로 구분

메모:

- 서비스 패턴 요약 페이지에서 각 항목은 현재 `service_xx_01.html`의 개요 페이지로 연결됩니다.
- 즉, 스킬에서 서비스 패턴을 참고할 때는 우선 개요 페이지를 시작점으로 삼고, 그 안의 `유형`, `이용 상황별 플로`, `체크리스트`를 읽는 흐름이 적절합니다.

## 3. 기본 패턴 상세 페이지 구조

샘플: 개인 식별 정보 입력  
출처: <https://www.krds.go.kr/html/site/global/global_01.html>

### 3.1 구조

기본 패턴 상세 페이지는 대체로 다음 요소를 가집니다.

- 유형
- 사용성 가이드라인
- 예시
- 접근성 가이드라인
- FAQ
- 변경 내역

### 3.2 개인 식별 정보 입력 문서에서 뽑은 실무 규칙

#### 공통

- 정말 필요한 개인정보인지 먼저 검토
- 왜 필요한지 명확히 설명
- 복사/붙여넣기 허용
- 기본값 지정 금지
- 플레이스홀더는 실제 입력값과 명확히 구분

#### 이름

- 모든 문자, 기호, 공백 입력 지원
- 한글 이름 기준으로 최소 3글자 이상 강제 같은 과한 제약 금지
- 여러 개 필드로 이름을 강제 분리하지 않음
- 개명 사용자 대안 제공
- 외국인 이름은 길이/공백/하이픈/대소문자 보존 고려
- 단일 입력 필드는 최소 320~360px 이상 확보 권장

#### 생년월일

- 날짜 선택기 사용 금지
- 연도/일을 셀렉트로 제공하지 않음
- 입력 형식을 명확히 안내

#### 성별

- 특정 성별 기본 선택 금지
- 필요하지 않다면 필수 입력에서 제외
- `선택 안 함` 옵션 제공 권장

#### 전화번호

- 목적에 맞는 전화번호 유형만 요청
- 명확한 레이블 제공
- 공백, 하이픈, 대시 등 다양한 입력 형식 허용
- 국가 코드는 콤보박스 제공
- 여러 번호를 요청하면 번호별 레이블 구분
- 주 연락 번호 지정 옵션 제공 가능

#### 접근성

- 입력 필드 목적에 맞는 `autocomplete` 속성 설정

스킬 반영 포인트:

- 개인정보 관련 UI는 단순 폼 설계가 아니라 입력 제약/복사 허용/포맷 가이드/선택 안 함 옵션까지 판단해야 합니다.

## 4. 서비스 패턴 상세 페이지 구조

샘플: 방문  
출처: <https://www.krds.go.kr/html/site/service/service_01_01.html>

### 4.1 구조

서비스 패턴 상세 페이지는 대체로 다음 순서를 가집니다.

- 유형
- 이용 상황별 플로(Flow)
- 사용성 가이드라인 체크리스트
- 적용 수준 필터
- 변경 내역

즉, 컴포넌트/기본 패턴보다 `사용자 여정`과 `적용 수준`이 중심입니다.

### 4.2 방문 패턴에서 확인된 유형

- 정책 및 서비스 정보
- 기관 정보
- 내비게이션
- 캠페인 및 홍보 정보
- 긴급 알림
- 연락처

### 4.3 방문 패턴의 플로

- 정보 탐색
- 정보 확인
- 이동

이 구조는 서비스 패턴 전반에서 스킬 워크플로우의 뼈대로 쓰기 좋습니다.

### 4.4 방문 패턴의 체크리스트 예시

문서에 노출된 항목 예:

- 필수:
  - 시급하거나 중요한 정보를 빠르게 인지 가능하게 제공
  - 최신의 정확한 정보 제공
- 권장:
  - 다양한 콘텐츠 구성 요소를 일관성 있게 구성
  - 사용자 데이터 기반 우선순위에 따라 배치
  - 사용자 관점에서 같은 목적 콘텐츠를 섹션 단위로 묶기
  - 방문 목적/사용자 유형별 핵심 정보 탐색 영역 구성
  - 정보 구분과 이동 방법을 명확하게 제시
  - 콘텐츠/서비스 명칭은 친숙하고 일관되게 유지
- 우수:
  - 로그인 기반 개인화된 바로가기 영역 배치

스킬 반영 포인트:

- 서비스 패턴 결과물은 “보여주기 좋은 화면”보다
  `필수`, `권장`, `우수`를 어느 수준까지 달성했는지 평가해야 합니다.

## 5. 기본 패턴과 서비스 패턴의 차이

### 기본 패턴

- 반복되는 UI 과업 묶음
- 입력/확인/오류/목록 같은 실무형 UI 블록
- 컴포넌트 조합 가이드 성격이 강함

### 서비스 패턴

- 서비스 전체 여정/핵심 과업 중심
- 정보 구조, 콘텐츠 우선순위, 사용자의 목적을 다룸
- 적용 수준 구분이 훨씬 중요

실제 스킬에서는 두 층을 분리해야 합니다.

- 서비스 수준 문제:
  서비스 패턴으로 해결
- 화면 조합 문제:
  기본 패턴으로 해결

## 6. 스킬로 옮길 때 필요한 판단 순서

1. 이 요청이 서비스 전체 플로 문제인가, 특정 화면 과업 문제인가
2. 특정 화면 과업 문제라면 어떤 기본 패턴에 해당하는가
3. 거기서 필요한 컴포넌트는 무엇인가
4. 접근성/적용 수준 체크는 어느 수준까지 필요한가

## 7. 스킬에 넣을 만한 패턴 선택 질문

### 서비스 패턴용 질문

- 사용자가 처음 방문해서 무엇을 해야 합니까
- 정보를 찾는 과정이 핵심입니까, 신청/로그인이 핵심입니까
- 중요한 정보는 무엇이며 즉시 보여야 합니까
- 필수/권장/우수 중 어느 수준까지 맞출 것입니까

### 기본 패턴용 질문

- 입력인가, 확인인가, 오류 처리인가
- 목록을 좁혀 찾는 문제인가
- 개인정보나 민감 정보 입력이 들어갑니까
- 사용자에게 즉시 피드백을 받아야 합니까
